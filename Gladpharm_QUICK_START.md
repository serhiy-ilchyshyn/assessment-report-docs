## 🎯 Success Criteria for Each Interview

### Interview 1: Project Sponsor + Finance Director ✓ Complete When:
- [ ] Business overview documented (company size, market position)
- [ ] Top 3-5 pain points identified
- [ ] DWH objectives and success metrics defined
- [ ] Project timeline and budget captured
- [ ] Key stakeholders identified

### Interview 2-4: Business Unit Heads ✓ Complete When:
- [ ] Top 5-10 metrics/KPIs per unit documented with definitions
- [ ] Current calculation methods captured
- [ ] User groups and usage patterns identified
- [ ] Desired future state understood

### Interview 5: IT Infrastructure ✓ Complete When:
- [ ] All source systems inventoried (7+ systems documented)
- [ ] Current DWH/BI platform and user base documented
- [ ] Infrastructure capacity (compute, storage, network) captured
- [ ] Hosting model (on-prem, cloud, hybrid) confirmed
- [ ] DR/backup strategy documented

### Interview 6: Data Owner & Security ✓ Complete When:
- [ ] Data quality issues and impacts documented
- [ ] Security requirements (RLS, CLS, encryption) identified
- [ ] Compliance requirements (GDPR, HIPAA, etc.) captured
- [ ] Data classification scheme confirmed
- [ ] Retention/deletion policies documented

### Interview 7: Database Admin & ETL ✓ Complete When:
- [ ] Loading strategies for each major system defined
- [ ] Current ETL/integration tools documented
- [ ] Data volumes and growth rates captured
- [ ] SLA requirements and peak load periods identified
- [ ] Error handling and monitoring procedures documented

### Interview 8: Business Analysts & Dimensions ✓ Complete When:
- [ ] Analytical dimensions identified (10+)
- [ ] Dimension hierarchies documented
- [ ] Slowly Changing Dimension (SCD) requirements identified
- [ ] Key drill-down paths understood
- [ ] Ad-hoc analysis patterns captured

## ⚠️ Common Pitfalls to Avoid

| Pitfall | Impact | Prevention |
|---------|--------|-----------|
| Skipping kickoff meeting | Stakeholder misalignment | Schedule formal kickoff week 1 |
| Incomplete systems inventory | Missing data sources | Have IT director verify all systems |
| Vague metric definitions | Poor requirements | Use metric template, get owner sign-off |
| Not documenting current state | Can't measure improvement | Capture baseline metrics |
| Insufficient security requirements | Non-compliant solution | Document ALL compliance needs upfront |
| Postponing document collection | Report delays | Request all docs at week 2 kickoff |
| Not validating findings | Incorrect assessment | Schedule review meetings week 4 |

## 📞 If You Get Stuck

### "Stakeholder won't commit to interview time"
→ Use Gladpharm_Data_Gathering_Strategy.md to show business case; offer 30-min version

### "Interviewee doesn't know the answer"
→ Ask them to identify who does; schedule follow-up with that person

### "Interview responses are vague"
→ Use the "Can you give me an example?" follow-up; reference Yuria-Pharm report as comparison

### "Too many systems to inventory"
→ Prioritize by data size/criticality; create subsystem detail sheets for complex ones

### "Data quality is worse than expected"
→ Document issues in detail; include in report as risk/recommendation

### "Compliance requirements change during interviews"
→ Capture all requirements; escalate to sponsor if scope increases
